# User Experience & Intelligent Assistance Patterns

## グループ概要

**User Experience & Intelligent Assistance Patterns** は、エンドユーザーと LLM のインタラクション品質を最大化し、自然で一貫性ある対話体験を提供するための設計指針です。ユーザー文脈の理解、先回り支援、説明責任の確保を通じて **満足度・信頼性・リテンション** を高めます。

### 主なユースケース
| ユースケース | 目的 | 代表パターン |
|--------------|------|--------------|
| カスタマーサポートボット | 問い合わせ意図を正確に把握 | Interactive Disambiguation |
| 生産性支援ツール | 次の操作を予測し提案 | User Action Prediction |
| ラーニングコーチ AI | 学習進捗を記憶し継続的指導 | Memory‑Augmented Interaction |
| 企業向けレポート生成 | 出力根拠の提示で信頼性確保 | Explainable AI Pattern |
| モバイルチャットアプリ | 高負荷時でも応答保証 | Fallback Model Selection |

---

## パターン詳細

> **記法**  
> • **説明**＝パターン概要 • **用途**＝典型シナリオ  
> • **解決課題**＝何を改善するか • **対象システム**＝想定プロジェクト  
> • **メリット**＝採用効果 • **注意点**＝設計時の留意点

### 1. Interactive Disambiguation Pattern
| 項目 | 内容 |
|------|------|
| **説明** | ユーザー入力が曖昧な際に補足質問を行い、意図を明確化してから回答する。 |
| **用途** | FAQ ボット、ナレッジ検索。 |
| **解決課題** | 誤解による不適切回答。 |
| **対象システム** | カスタマーサポート、予約システム。 |
| **メリット** | 正答率・CSAT 向上。 |
| **注意点** | 質問が多すぎると UX 劣化。 |

---

### 2. User Action Prediction Pattern
| 項目 | 内容 |
|------|------|
| **説明** | ユーザー行動ログを解析し、次に行う可能性が高い操作を予測して提案・自動化。 |
| **用途** | ドキュメント編集、E コマース。 |
| **解決課題** | 多クリック操作・入力負荷。 |
| **対象システム** | 生産性 SaaS、業務支援ツール。 |
| **メリット** | 操作短縮、エンゲージメント向上。 |
| **注意点** | 誤提案はユーザー不快感を招く。 |

---

### 3. Memory‑Augmented Interaction Pattern
| 項目 | 内容 |
|------|------|
| **説明** | ユーザープロファイル・対話履歴を長期メモリとして保持し、次回以降の会話に反映。 |
| **用途** | 学習支援、パーソナライズドコーチ。 |
| **解決課題** | 毎回コンテキストを再入力する煩雑さ。 |
| **対象システム** | 継続利用アプリ、健康管理 AI。 |
| **メリット** | 継続性・一貫性ある UX。 |
| **注意点** | プライバシー同意とデータ保持ポリシー。 |

---

### 4. Explainable AI Pattern for LLM
| 項目 | 内容 |
|------|------|
| **説明** | LLM の回答根拠や参照ソースをユーザーが理解しやすい形で提示。 |
| **用途** | 高信頼性要求、規制産業。 |
| **解決課題** | ブラックボックス出力への不信感。 |
| **対象システム** | 金融アドバイス、法務レビュー。 |
| **メリット** | 透明性・コンプライアンス向上。 |
| **注意点** | 説明が冗長・難解にならないよう要約技法必須。 |

---

### 5. Fallback Model Selection Pattern
| 項目 | 内容 |
|------|------|
| **説明** | 高性能モデルが失敗・遅延時に、簡易モデルやキャッシュ応答へフェイルオーバ。 |
| **用途** | 高可用性チャット、オフライン対応。 |
| **解決課題** | モデル障害・レイテンシ過大。 |
| **対象システム** | モバイルチャット、IoT デバイス。 |
| **メリット** | サービス停止回避、UX 維持。 |
| **注意点** | モデル間で語調・内容を揃える正規化層。 |

---

## アンチパターン

### 1. Uncontextualized Responses（文脈無視応答）
- **概要**: 過去対話やユーザ情報を無視し毎回初対面のような回答をする設計。
- **問題点**: UX が機械的・一貫性欠如、再入力負担。
- **発生例**: チャット再読み込みで毎回名前を尋ねられユーザ不満。

### 2. Over‑Autonomy（過剰自律応答）
- **概要**: AI がユーザ確認なしに推測で行動・購入などを実行。
- **問題点**: 主体性喪失、不快・危険行動を招く。
- **発生例**: AI コーチが勝手にスケジュール変更→ユーザ混乱。
